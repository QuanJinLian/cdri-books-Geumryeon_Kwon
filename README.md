## 2. 실행 방법 및 환경 설정

#### ⚙️ 환경 변수 설정 (.env)
루트 디렉토리에 `.env` 파일을 붙여놓아야 합니다.

#### 🚀 설치 및 실행
```bash
# node -v 통일 ,  .nvmrc 참고
# Now using node v23.11.1 (npm v10.9.2)
nvm use

# 의존성 설치
npm ci

# 로컬 서버 실행 
npm run dev
```

실행 주소: http://localhost:5173/


## 3.폴더 구조 및 주요 코드 설명

```
src/
 ├── assets/              #  scss, image 와 같은 파일
 ├── components/          #  공통 컴포넌트
 ├── defines/             #  상수, 고정된 설정 값 등
 ├── hooks/               #  공용 커스텀 훅
 ├── pages/               #  페이지 관련된 컴포넌트 (api 요청)
 ├── router/              #  router config
 ├── types/               #  공통 타입 모음
 ├── util/                #  공통, global 함수 모음 ex) nanoid, axios
 └── App.tsx              #  
```

#### 직접 구현한 컴포넌트
- `components/card` 
  - `BookCard` - 책 카드 리스트 
    - 주요 기능: 카드 리스트중 상세 펼침 카드 아이디 리스트 관리
  - `BookCardItem` - 책 카드 싱글
    - 주요 기능: 한 카드의 마크업 구조
  - `BookImg` - 책 카드에서 사용될 하트 버튼을 포함한 img
      - 주요 기능: 하트 버튼 없이 단독 img 로도 사용 가능 
  - .....
- `components/empty`
  - `Empty` - 다양한 (img) + (텍스트) 상황에서 사용 가능
- `components/input`
  - `SearchInput` - 다양항 (input) + (힌트(검색어)) 상황에서 사용가능
    - 주요 기능: 
      - input 밑에 연관검색어 혹은 검색기록와 같은 항목들 뜨게 할 수 있음 
      - input debounce 와 같은 작업은 밖에서 하게 설계 하였으므로 확장성이 강함
      - 검색어 옆에 x 버튼 다른 아이콘 혹은 문자로 커스텀 가능하게 설계
      - 혹시 custom component 렌더링 원하면 해당 부분 뚫어주면 됨(`src/components/hint/HintItem.tsx`)
- `components/number` 
  - `ItemCount` - 다양한 (텍스트) + (prefix)(숫자)(suffix) 상황에서 사용가능
    - 주요 기능: 숫자 기본 콤마 처리, 다른 형대의 converting 을 원하면 함수를 넘겨주면 됨
    - 사용예제:
      - "도서 검색 결과 총 7,000건"
      - "책이름 작가이름 7,000원"
      - "원가 7,000원"
- `components/scroll`
  - `InfiniteScroll` - 무한 스크롤를 위한 공용 컴포넌트
    - 주요 기능: 특정 컴포넌트 전부 가시 상황일 경우 next 함수 실행


#### MUI 랩핑한 컴포넌트
- `components/popover` - 상세 검색 팝업 및 검색 이력 리스트 표출을 위해 필요
- `components/select`  - 상세 검색 타겟 선택 시 필요
- `components/tabs`    - 도서 검색 메뉴와 내가 찜한 책 메뉴 토글을 위해 필요


#### SCSS 파일 기준
- `components/*` 안에 있는 경우:  제가 직접 만든 컴포넌트 중 라이브러리 성격(나중에 따로 떼서 피키징할 수 있는)
- `components/assets/style` 에 있는 경우: 공통 설정, page 관련 설정, 프로젝트 특정 설정

## 4.라이브러리 선택 이유

| 라이브러리          | 선택 이유                                                           |
| -------------- | --------------------------------------------------------------- |
| TanStack Query | 서버 상태 관리의 복잡성을 줄이고, 캐싱 및 무한 스크롤 구현에 최적화된 인터페이스를 제공을 위해 선택하였습니다. |
| axios          | 인터셉터를 통한 헤더 관리 와 에러 핸들링이 간편하여 사용하였습니다.                          |
| sass           | 평소에 sass 로 주로 스타일링 작업을 하여 선택하게 되었습니다.                           |
| mui            | 비교적 복잡한 tabs, popover 와 같은 컴포넌트는 개발하기에는 시간이 부족하여 선택하게 되었습니다.    |
| react-hook-for | 입력 폼 상태관리에 용이하여 선택하게 되었습니다.                                     |

## 5. 강조하고 싶은 기능
- 복잡한 데이터 머지 로직
    - 서버에서 가져온 데이터와 로컬에 저장된 찜리스트를 결합하여, 각 도서 아이템에 실시간으로 "찜"여부를 주입하는 로직 구현하였습니다. 이렇게 함수를 두개 나눠서 작업하면 나중에 유지보수 하기에도 용이하다고 생각합니다.
- 초소 단위로 설계된 컴포넌트와 사용하기 쉽게 랩핑한 UI 컴포넌트
    - 각 컴포넌트 설계 시 최소 단위로 하여 쪼개서 설계하였고, 최대한 유연한 사용을 위해 Props 설계를 하였습니다. 이는 나중에 재조립, 재사용에 용이할 것으로 예상됩니다.
    - MUI 컴포넌트는 기능이 많고 아주 좋으나 매번 조립해서 쓰기에는 설정할게 많고 코드도 더러워 집니다. 그리하여 프로젝트에 맞게 미리 랩핑 시키면 간단한 설정 값으로 바로 쓸 수 있어 생산성이 올라갑니다.
- Flexbox 기반 스타일링
    - `flex: 1` 를 이용하여 다양한 해상도에서도 검색영역은 나머지 전체 영역을 차지하게 함
- Card 상세보기 애니메이션
    - 상세보기 버튼 토글 시 이미지와 카드 높이가 자연스럽게 애니메이션 됨
